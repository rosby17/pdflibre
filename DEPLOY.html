Architecture

Vue d'ensemble

ℹ️

Pas de base de données requise pour le MVP. Les fichiers traités vivent en mémoire/tmp pendant 1h puis sont supprimés automatiquement. Aucune donnée utilisateur n'est persistée.

📈 Scalabilité

Stratégie de montée en charge

Le backend est conçu pour évoluer sans réécriture. Voici les 3 paliers :

🚀 Maintenant (MVP)
Railway Starter gratuit
1 instance · 512MB RAM
Asyncio job queue
Fichiers en /tmp
~50 req/jour
📦 Croissance
Railway Pro (~$5/mois)
Auto-scale 2–4 instances
Redis + Celery workers
Cloudflare R2 (stockage)
~5 000 req/jour
🏭 Production
Kubernetes ou Fly.io
N workers indépendants
S3 / R2 + CDN
Supabase auth
Illimité
💡

Le code actuel est déjà prêt pour le palier 2 : il suffit de swapper app/jobs/queue.py par une implémentation Redis sans toucher aux routers.

✓

🐙 Étape 1

Créer le repo GitHub

1

Créer un nouveau repo

Sur github.com → New repository → nom : pdflibre → Public ou Private → Create.

2

Dézipper et pusher le projet

bash
unzip pdflibre-full.zip
cd pdflibre
git init
git add .
git commit -m "feat: initial PDFLibre MVP"
git remote add origin https://github.com/VOTRE_USER/pdflibre.git
git push -u origin main
3

Vérifier la structure

Le repo doit contenir backend/, frontend/, docker-compose.yml, README.md.

✅

Railway et Vercel vont détecter automatiquement la configuration grâce aux fichiers railway.json et la structure du projet.

🚂 Étape 2

Déployer le backend sur Railway

Railway est la plateforme la plus simple pour déployer du Python/Docker gratuitement.

1

Créer un nouveau projet

Sur railway.app → Dashboard → New Project → Deploy from GitHub repo → sélectionnez pdflibre.

2

Configurer le Root Directory

Railway va détecter tout le repo. Vous devez lui dire que le backend est dans un sous-dossier :

Settings → Source → Root Directory → saisir : backend

⚠️

Sans ce paramètre, Railway cherchera le Dockerfile à la racine et échouera.

3

Ajouter les variables d'environnement

Settings → Variables → ajouter :

VariableValeurNote
DEBUGfalseDésactive les docs Swagger en prod
ALLOWED_ORIGINS["https://pdflibre.vercel.app"]À mettre après avoir l'URL Vercel
MAX_FILE_SIZE_MB100Limite par fichier
RATE_LIMIT_RPM30Requêtes/min par IP
MAX_CONCURRENT_JOBS4512MB RAM → max 4
ℹ️

Railway injecte $PORT automatiquement. Le Dockerfile l'utilise déjà.

4

Lancer le déploiement

Cliquez Deploy. Railway va builder l'image Docker (~3 minutes la première fois, poppler est lourd).

Suivez les logs en temps réel dans l'onglet Deployments.

5

Récupérer l'URL publique

Settings → Networking → Generate Domain → vous obtenez quelque chose comme :
https://pdflibre-api-production.up.railway.app

Testez : https://votre-url.railway.app/health → doit retourner {"status":"ok"}

✅

Backend en ligne ! Notez l'URL — vous en avez besoin pour l'étape Vercel.

▲ Étape 3

Déployer le frontend sur Vercel

1

Modifier l'URL de l'API dans le frontend

Dans frontend/index.html, trouvez la ligne API_BASE et remplacez par votre URL Railway :

javascript
// Avant :
const API_BASE = window.location.hostname === 'localhost' ? ...

// Après :
const API_BASE = 'https://pdflibre-api-production.up.railway.app';

Commitez et pushez ce changement :

bash
git add frontend/index.html
git commit -m "config: set production API URL"
git push
2

Importer le projet sur Vercel

Sur vercel.com → Add New Project → importer pdflibre depuis GitHub.

Configuration :

ParamètreValeur
Framework PresetOther
Root Directoryfrontend
Build Command(laisser vide)
Output Directory.
3

Déployer

Cliquez Deploy. Vercel déploie en ~30 secondes. Vous obtenez une URL comme :
https://pdflibre.vercel.app

4

Mettre à jour ALLOWED_ORIGINS sur Railway

Retournez sur Railway → Variables → mettez à jour :

env
ALLOWED_ORIGINS=["https://pdflibre.vercel.app"]

Railway redéploiera automatiquement.

🎉

Tout est en ligne ! Testez un outil PDF sur votre URL Vercel.

🌐 Domaine custom

Ajouter votre propre domaine

Frontend (Vercel)

1

Vercel → Settings → Domains → Add

Saisissez pdflibre.com (ou votre domaine).

2

Configurer le DNS

Chez votre registrar, ajoutez un enregistrement CNAME :
www → cname.vercel-dns.com
ou A record pour l'apex.

Backend (Railway)

1

Railway → Settings → Networking → Custom Domain

Saisissez api.pdflibre.com.

2

Configurer le DNS

CNAME : api → votre-service.railway.app

3

Mettre à jour API_BASE dans le frontend

js
const API_BASE = 'https://api.pdflibre.com';
💻 Dev local

Lancer en local

Option A — Docker (recommandé)

bash
cd pdflibre
docker-compose up --build
# Frontend → http://localhost:3000
# API docs → http://localhost:8000/docs  (DEBUG=true)
# Health   → http://localhost:8000/health

Option B — Python natif

bash
# Installer poppler
# Ubuntu/Debian:
sudo apt install poppler-utils
# macOS:
brew install poppler

# Backend
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
DEBUG=true uvicorn app.main:app --reload --port 8000

# Frontend — dans un autre terminal
cd ../frontend
python -m http.server 3000

Fichier .env local

bash
cp backend/.env.example backend/.env
# Éditez backend/.env et mettez DEBUG=true
📊 Monitoring

Surveiller votre app

Railway (built-in)

Railway affiche CPU, RAM, et logs en temps réel dans le dashboard. Configurez des alertes email dans Settings → Notifications.

Health check endpoint

bash
# Réponse attendue :
curl https://votre-url.railway.app/health

{
  "status": "ok",
  "version": "1.0.0",
  "max_file_mb": 100,
  "rate_limit_rpm": 30
}

Uptime monitoring gratuit

Utilisez UptimeRobot (gratuit) pour monitorer /health toutes les 5 minutes et recevoir des alertes.

⚠️

Railway Starter gratuit : l'app se met en veille après 30 minutes d'inactivité. Le premier appel après inactivité sera lent (~5s cold start). Pour éviter cela : Railway Pro ($5/mois) ou pinger /health toutes les 10min avec UptimeRobot.

📈 Passer à l'échelle

Quand vous grandissez

Ajouter Redis pour les jobs (>500 users/jour)

bash
# Railway → Add Plugin → Redis (gratuit 25MB)
# Railway injecte automatiquement REDIS_URL

pip install redis celery

Swappez app/jobs/queue.py par une implémentation Celery — les routers n'ont pas besoin de changer.

Stockage persistant (fichiers > 1h)

bash
pip install boto3
# Cloudflare R2 : 10GB gratuit, compatible S3
# Variable : R2_BUCKET, R2_ACCESS_KEY, R2_SECRET_KEY

Auth utilisateurs (historique)

Supabase (PostgreSQL + Auth) est gratuit jusqu'à 500MB. SDK Python disponible.
